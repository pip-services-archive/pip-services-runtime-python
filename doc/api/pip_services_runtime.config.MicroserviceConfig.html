class pip_services_runtime.config.MicroserviceConfig.MicroserviceConfig(content=None)¶

Bases: object

Configuration for the entire microservice. It can be either stored in JSON file on disk, kept in remote configuration registry or hardcoded within test.

static from_tuples(*tuples)¶

Creates component configuration using hardcoded parameters. This method of configuration is usually used during testing. The configuration is created with ‘Undefined’ category since it’s not used to create a component.

Args:
tuples: configuration parameters as <key> + <value> tuples

Returns: constructed microservice configuration

static from_value(value)¶

Creates microservice configuration using free-form objects.

Args:
value: a free-form object

Returns: constructed microservice configuration

get_raw_content()¶

Gets the raw content of the configuration as dynamic map Returns: DynamicMap with all microservice configuration parameters.

get_section(category)¶

Gets configurations of components for specific section.

Args:
category: a category that defines a section within microservice configuration

Returns: ComponentConfig array with components configurations

remove_sections(*categories)¶

Removes specified sections from the configuration. This method can be used to suppress certain functionality in the microservice like api services when microservice runs inside Lambda function.

Args:
categories: a list of categories / section names to be removed.

Returns: None
